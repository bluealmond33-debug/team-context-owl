# NotebookLM Handoff — [프로젝트] — YYYY-MM-DD

> **용도:** GitHub/Obsidian의 변경을 NotebookLM에서 질문·팀 브리핑·슬라이드 생성에 쓸 수 있는 하나의 읽기용 소스로 정리한다.
>
> **중요:** 이 파일은 NotebookLM과 자동 동기화되지 않는다. 업로드 전에는 아래 범위와 민감정보 점검을 사람이 확인한다.

## 0. 이 패키지의 기준

- **기준 브랜치 / 커밋:** `[main / abc1234]`
- **비교 범위:** `[지난 패키지 커밋] → [현재 커밋]`
- **작성일:** `[YYYY-MM-DD]`
- **대상:** `[팀 브리핑 / 온보딩 / 발표 자료 / 의사결정 검토]`
- **포함한 폴더·문서:** `[경로와 링크]`
- **제외한 내용:** `[raw 회의록 / 오래된 초안 / 비밀·개인정보 가능 자료]`

## 6. NotebookLM에서 바로 쓸 질문

### 팀 브리핑
> 이 소스만 근거로, 확정된 결정·이번 변경·미확정 토의를 구분한 3분 팀 브리핑을 한국어로 작성해줘. 추측은 넣지 말고, 각 핵심 주장 뒤에 근거 문서명을 적어줘.

### 발표 슬라이드
> 이 소스만 근거로 팀 공유용 슬라이드 초안을 만들어줘. 확정 사항은 핵심 메시지로, 미확정 사항은 마지막의 토의 질문으로 분리해줘. 사실로 검증되지 않은 수치·성과·연동은 만들지 마.

### 온보딩 / Q&A
> 새 팀원이 이해해야 할 프로젝트 목표, 현재 상태, 확정된 결정, 다음 행동을 순서대로 설명해줘. 오래된 내용이나 미확정 가설은 확정 사실처럼 말하지 마.

### 의사결정 검토
> 이 소스에서 아직 결정되지 않은 항목만 표로 정리해줘. 각 항목에 현재 선택지, 이미 있는 근거, 추가로 필요한 검증을 넣어줘.

## 업로드 전 확인

- [ ] 기준 커밋과 비교 범위를 적었다.
- [ ] API 키·토큰·개인정보·실고객 데이터·비공개 계약 내용을 제외했다.
- [ ] 오래된 결정과 현재 결정을 혼합하지 않았다.
- [ ] 원문 근거 링크를 남겼다.
- [ ] NotebookLM에 올릴 목적에 맞지 않는 raw 자료를 제외했다.
- [ ] 업로드 후에는 이 파일의 기준 커밋을 NotebookLM 노트에 기록한다.
